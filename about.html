About me

Senior Software Engineer · MSc Artificial Intelligence candidate.
Johannesburg, South Africa.

Profile

Senior backend-leaning full-stack engineer with about a decade of production experience across permanent, contract, and remote-first roles. Recent focus on scalable distributed systems on Azure — microservices, event-driven integrations, and Infrastructure-as-Code — for public-sector healthcare workloads with hard availability, auditability, and data-protection constraints.

The MSc AI is the formal track for what has become an increasingly AI-centred practice: I'm Microsoft Azure AI Engineer Associate certified, work daily with AI-assisted development tooling, and want a deeper grounding in the maths, methods, and ethics underneath the tooling so I can lead AI-system design rather than only integrate AI features.

Questions I want the programme to help me answer:

  • How do I architect AI-augmented systems where bias, accountability, and patient safety are non-negotiable?
  • Where is the line between AI-assisted clinical decision support and AI-driven diagnosis, and how does software architecture enforce it?
  • What does responsible MLOps look like for regulated public-sector workloads?

Experience

Senior Software Engineer — South Africa Western Cape Government, Department of Health (Aug 2022 – Dec 2025, contract, remote). Lead designer of mission-critical healthcare microservices in C#/.NET 6+ on Azure SQL, serving 100K+ daily users at 99.9% availability. RESTful and GraphQL APIs via Azure API Management; Angular/TypeScript front ends for clinical users; Azure-native CI/CD on Azure DevOps with ARM-template IaC and Application Insights observability.

Software Engineer — EOH Holdings (Aug 2021 – Jul 2022, contract, remote). Distributed enterprise systems in C#/.NET Core/SQL Server with Angular front ends; REST API ecosystems; CI/CD on Azure DevOps and GitLab CI; code-review leadership.

Full Stack Developer — Afriword Digital (Feb 2016 – Jul 2021, permanent, hybrid). C#/.NET Framework + SQL Server applications and REST APIs; front-end progression into TypeScript and Angular. Where the production engineering practice was formed.

What this e-Portfolio is for

UoEO requires every Computing-route MSc student to maintain a GitHub Pages e-Portfolio that demonstrates their understanding of module content, ability to draw connections between modules and coursework, and capacity for self-assessment with supporting evidence. It is the artefact through which the programme is ultimately reviewed and through which submissions for individual modules are made.
